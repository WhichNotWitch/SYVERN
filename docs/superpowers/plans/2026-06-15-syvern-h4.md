# SYVERN H4 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add deterministic H4 anti-gaming veto hardening and IPT consistency evaluation for eligible `full` validation requests.

**Architecture:** Keep anti-gaming rules in Stage 3 and hard veto decisions in `veto.py`, then add a focused `ipt.py` module that runs exact H3 structural matching over caller-supplied perturbation texts. Pass perturbations through the API/cache/pipeline path, evaluate IPT only after T0, constraint, veto, and Stage 4 pass, and keep online reward/data filter paths unevaluated.

**Tech Stack:** Python 3.11+, FastAPI, Pydantic v2, pytest, current `src/syvern` package layout.

```python
from __future__ import annotations

from typing import Any

from syvern.adapters.stub import extract_element_summary
from syvern.settings import SyvernSettings
from syvern.structural import match_structural


def evaluate_ipt(
    *,
    perturbations: list[str] | None,
    reference: dict[str, Any] | None,
    settings: SyvernSettings,
) -> bool | None:
    if not perturbations:
        return None

    for perturbation in perturbations:
        summary = match_structural(
            extract_element_summary(perturbation),
            reference,
            settings,
        )
        if summary.f1 < settings.ipt_threshold:
            return False
    return True
```
